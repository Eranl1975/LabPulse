# Coding Rules

- Use TypeScript everywhere
- Keep modules small and single-purpose
- Prefer explicit types
- Prefer rule-based logic in MVP
- Avoid hidden magic and silent fallbacks
- Keep functions testable
- Add source traceability to evidence-related outputs
- Do not hardcode secrets
- Do not overengineer abstractions
- Write brief comments only where needed
- Do not rewrite unrelated files
- Keep summaries short
- If integration is unavailable, create an adapter interface only
- State uncertainty explicitly
- Keep markdown files as short as possible
- Prefer minimal modular code over large frameworks
